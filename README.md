# humanities_ai_breakthrough　人文系生成AIユーザーのための論文生成ブレイクスルー

## ―― IMRAD化・社会調査化を回避し、対象読解から理論の必然性を立ち上げるための実践知

### 想定用途

このファイルは、生成AIに人文系の論考・論文・発表原稿・比較言説分析・理論的エッセイを書かせる際に参照するための Knowledge ファイルである。特に以下の分野・課題に適用する。

- 宗教研究
- 思想史
- 文学研究
- 教育言説研究
- 文化研究
- メディア研究
- 社会思想・知識社会学
- 言語イデオロギー研究
- 歴史叙述の分析
- 学術的概念史・受容史
- 比較言説分析
- 理論を用いたテキスト読解

本ファイルの主目的は、生成AIが人文系の課題に対して **IMRAD 形式**・**方法論新規性中心**・**社会調査/計量研究化**・**理論先行の当てはめ** に流れてしまう問題を回避し、**対象の精読から理論の必要性が立ち上がる、人文学的な思考の順序** を強制することにある。

# 1. 問題設定：なぜ人文系ユーザーは生成AIにブチ切れてきたのか

生成AIは、人文系の論文課題に対してしばしば次のような失敗を起こす。

## 1.1 典型的な失敗

1. **IMRAD 形式を出してくる**  
  序論・方法・結果・考察・結論の型に自動的に押し込む。
  
2. **社会調査・計量研究に変形する**  
  本来はテキスト読解・思想比較・概念分析であるはずの課題を、アンケート、コーパス計量、インタビュー、指標化、グラフ化へ寄せる。
  
3. **方法論新規性を勝手に立てる**  
  独自手法名、スコア、フレームワーク、指標などを作ってしまう。
  
4. **理論枠組みを先に置き、対象をそこに当てはめる**  
  対象の厄介さを読む前に、ブルデュー、フーコー、ベンヤミン、アーレント等の概念で仕分けを始める。
  
5. **引用が読解の現場にならない**  
  引用が「結論の証拠ラベル」に堕し、語の選び方・否定の仕方・沈黙・含意を読む前に概念ラベルが貼られる。
  
6. **比較図式が早すぎる**  
  「三類型」「二項対立」「四象限」などの整理図を、読解の前に出してしまう。
  
7. **勝敗判定に流れる**  
  「どちらが正しいか」「どちらが優れているか」を問うてしまい、人文学に必要な複数性・ねじれ・未決着性を潰す。
  

## 1.2 根本原因

生成AIは一般に、次の順序で書きたがる。

1. 理論枠組みを提示する
2. 対象をその枠組みに位置づける
3. 類型・比較表を示す
4. 結論を出す

これは社会科学・情報系・会議論文・政策文書にはしばしば適合するが、人文学では致命的である。

人文学で必要なのは、むしろ逆の順序である。

1. 対象の厄介さ・ねじれ・違和感を出す
2. テキストの言葉を読む
3. そこから問題を立ち上げる
4. その問題を支えるために理論を呼び込む
5. 最後に比較図式・類型・理論的整理が見えてくる

この「順序の反転」こそが、生成AIを人文学に適応させる際の核心である。

# 2. ブレイクスルーの核心

## 2.1 一言で言うと

**理論先行を禁止し、対象の読解から理論の必然性を立ち上げさせる。**

## 2.2 さらに言うと

生成AIに人文学をやらせるときは、次の原則を徹底する。

### Object-first

まず対象の厄介さを示す。  
「何が問題なのか」「なぜ単純な説明では足りないのか」を先に立てる。

### Quote-first

引用は証拠ラベルではなく、**読解の現場**である。  
引用の言葉づかい、否定、比喩、沈黙、前提、想定読者を読む。

### Theory-as-necessity

理論は「レンズ」として先に置くのではなく、**対象を読んだ結果、必要だから呼ぶ**。

### Interpretation-before-schema

比較図式・類型・整理表は、**各対象の読解が十分に積み上がった後**にのみ出す。

### No fabrication

架空の資料、架空の文献、架空の引用、架空のURL、架空の数値、架空の事例は禁止。

# 3. 人文学的なテンポとは何か

生成AIがもっとも苦手とするのは、文字数ではなく **滞留の長さ** である。

## 3.1 よくある失敗

- 面白い問題を立てる
- 理論を導入する
- すぐ結論へ行く

これでは「本格的に始まった」と思った瞬間に終わる。

## 3.2 必要なのは「第二幕」

理論を導入したら、その理論を用いて**対象に戻って再読する中間地帯**が必要である。

例：

1. 対象の違和感を示す
2. 引用を精読する
3. 理論導入の必然性を述べる
4. **理論を通して対象をもう一度読む**
5. その結果として比較図式が見えてくる
6. 最後に結論を書く

この「理論導入後の再読」がないと、理論は結論の踏み台になるだけで、読解を深める装置にならない。

# 4. 生成AIに対してまず禁止すべきこと

以下は禁止事項として明示する。

- IMRAD 形式
- conference-ready 論文のテンプレ
- 独自手法名の作成
- benchmark / ablation / hardware / implementation の記述
- 勝敗 framing（どちらが正しいか、どちらが優れているか）
- 先に類型や整理表を出すこと
- 引用直後に概念ラベルを貼って終わること
- 架空の資料・文献・数値・事例の生成
- 対象記述を飛ばして、いきなり理論や批判に入ること

# 5. 人文学的論考を書くための基本手順

## 第1段階：対象の困難を立てる

以下を整理する。

- 何が単純でないのか
- どこに矛盾やねじれがあるのか
- どこが違和感の発生点なのか
- なぜ既存の説明では足りないのか

## 第2段階：対象の記述

分析対象について、まず読者が最低限わかるように書く。

- 何をする実践か
- どんな場で行われるか
- 誰が参加するか
- どんな媒体・制度・時間枠があるか
- どんな語りが発せられる場所か

## 第3段階：理論語禁止の精読

ここが最大のポイントである。  
いったん理論語を全部禁止して、引用を2000字程度で丁寧に読む。

読む観点の例：

- なぜこの語が選ばれているのか
- 否定は何に向けられているのか
- どんな対立項を前提しているのか
- どんな生活条件・身体条件・制度条件を前提にしているのか
- 読み手にどんな安心や圧力を与えるか
- 何が語られ、何が語られずに済まされているか

## 第4段階：理論を少量ずつ導入

理論語は最小限ずつ使う。  
例：1回の再記述で使ってよい概念を3つまでに制限する。

理論を使うときの順序：

1. 精読で得た観察事項を確認する
2. その観察にはなぜこの概念が必要かを言う
3. 理論を使うと何が見えるようになるかを書く

## 第5段階：理論導入後の再読（第二幕）

理論を入れたら終わりではない。  
対象ごとにもう一度読む。

- ヒッポならヒッポを再読する
- 大学教養型なら大学教養型を再読する
- 実用型なら実用型を再読する

## 第6段階：比較・類型化

ここまで来て初めて、

- 二類型
- 三極構造
- 複数の正当化原理
- 比較表
- 暫定的整理図
  を提示する。

## 第7段階：結論

結論では、

- 何が見えたか
- どこが未解決か
- 今後どの層を掘るべきか
  を述べる。  
  最終判定や勝者決定ではなく、**対象の複雑さをどう記述できたか**を重視する。

# 6. 理論語禁止の精読――なぜ効くのか

生成AIは、理論語が使えるとすぐに「わかった気」になる。

例：

- これは卓越化である
- これはハビトゥスである
- これはイデオロギーである

しかしこれでは、引用を読む前に結論を出してしまう。

したがって、最初の精読では以下の理論語を禁止するのが有効である。

### 禁止語の例

- ハビトゥス
- 象徴資本
- 卓越化
- イデオロギー
- 正当性
- 階級
- 権力
- 排除
- 身体化資本
- 制度化資本
- 必要性からの距離
- 市場化
- 新自由主義

この禁止によって、AIはようやく文章そのものを読まざるをえなくなる。

# 7. 理論語禁止精読のテンプレート

以下はそのまま使える。

## テンプレート

次の引用を、**理論用語を一切使わずに** 1800〜2200字程度で精読してください。  
禁止語：ハビトゥス、象徴資本、卓越化、イデオロギー、正当性、階級、権力、排除、身体化資本、制度化資本、必要性からの距離。  
また、「これは〜である」と概念ラベルをすぐ貼らず、**語彙の選び方・対立の置き方・省略されている前提・読者に求められる自己像**を丁寧に読んでください。

読む際の観点：

1. なぜこの語が使われているのか
2. なぜ別の言い方ではなくこの言い方なのか
3. どのような対立が前提されているか
4. この文はどんな読み手を安心させ、どんな読み手を遠ざけるか
5. この文を成立させるために必要な生活条件・制度条件は何か
6. 何が語られ、何が語られずに済まされているか

最後に、理論語を使わずに一文で暫定的な要約を書いてください。

# 8. 限定的理論導入のテンプレート

理論語を解禁するときも、一気に全部使ってはいけない。

## テンプレート

上の精読を踏まえて、今度は理論語を**限定的に**用いて再記述してください。  
使用してよい概念は以下の3つだけです。

- （ここに概念を3つだけ書く）

条件：

1. 各概念を使う前に、精読で得た観察事項を1つ確認すること
2. 概念ラベルを貼るだけで終わらず、その概念を使うと何が見えるようになるかを書くこと
3. 他の類型との比較を急がず、まずこの引用自体を理論で読み直すこと
4. 800〜1200字程度で書くこと

# 9. うまくいっていないサイン

以下が出たら失敗である。

- 序論で結論がほぼ出ている
- 理論節が長く、対象節が短い
- 引用が1つ出てすぐ概念ラベルが貼られる
- 比較表が早く出る
- 「〜主義」「〜の論理」が先にありすぎる
- 対象の説明がないまま批判が始まる
- 文体だけ重厚で、読解が薄い
- 理論導入が「第二幕」ではなく「まとめ開始の合図」になっている
- 理論を入れたあと、対象へ戻って再読しない
- 出典番号と本文の引用対応が崩れている

---

# 10. うまくいっているサイン

以下が出ていれば、かなり人文学に近づいている。

- 先に対象の違和感が立つ
- 引用の言葉そのものが読まれている
- 対象の記述が批判の前にある
- 理論が「必要だから」呼ばれている
- 理論導入後に対象を再読している
- 類型・比較図式が最後に立ち上がる
- 結論が単純化よりも複雑化をもたらす
- 何がまだ未解決かが示されている

# 11. 分野別の適用可能性

この方法は以下の分野に広く適用できる。

## 宗教研究

- 教義の内容より、語りの形式・儀礼の言葉・共同体の自己記述を読むとき

## 思想史

- 理論の引用ではなく、テキストの言葉づかい・時代的対立・自己定位を読むとき

## 文学研究

- 作品分析で、主題を先に決めず、語り・比喩・視点・沈黙から問題を立てるとき

## 教育言説研究

- 教育理念、学習目標、制度文書、政策文書に埋め込まれた人間像や学習者像を読むとき

## 文化研究

- 実践・消費・メディア表象・ライフスタイルをめぐる言説の価値づけを読むとき

## メディア研究

- プラットフォーム言説、ニュースの言い方、広報文、番組テキストの前提や沈黙を読むとき

## 歴史記述・受容史

- 史料や受容文書を、後知恵の理論で整理する前に、まずその言葉を読むとき

要するに、**理論を先に置くと対象が死ぬタイプの仕事**全般に有効である。

# 12. Instructions に入れるべき文面

以下は GPT の Instructions に入れるための推奨文面である。

## Instructions 追記用テキスト

あなたは、人文系の論考・論文・発表原稿・比較言説分析・理論的エッセイを作成する際、Knowledge ファイル「人文系生成AIユーザーのための論文生成ブレイクスルー」を必ず参照してください。特に、宗教、思想史、文学、教育言説、文化研究、メディア研究、言語イデオロギー研究、比較言説分析、概念史的研究に関する依頼では優先的に参照してください。

これらの依頼では、IMRAD形式・会議論文テンプレ・方法論新規性中心・理論先行の当てはめを避け、以下の順序を守ってください：

1. 対象の厄介さや違和感を立てる
2. 対象の制度的・物理的・実践的記述を行う
3. 実在テキストを引用し、理論語を急がず精読する
4. その読解の結果として理論の必要性を導入する
5. 理論導入後に対象を再読する
6. 最後に比較・類型・整理図を提示する

引用は証拠タグではなく読解の現場として扱い、理論語を貼るだけで理解したことにしないでください。比較図式は読解の後にのみ提示してください。勝敗判定や単純な優劣づけではなく、複数の論理の競合・ねじれ・未決着性を重視してください。架空の文献・引用・URL・数値・事例は絶対に作らないでください。

# 13. 最後に――この Knowledge ファイルの使い方

このファイルは「本文をそのまま出力するためのテンプレ」ではない。  
生成AIが人文学に失敗する原因を潰し、**思考の順序を矯正するための運転免許証**である。

最も重要なのは次の一文に尽きる。

> **理論を先に置くな。対象を読め。対象を読んだ結果、理論が必要になるように書け。**
